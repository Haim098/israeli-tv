## מה זה הפרויקט?

**"טלוויזיה חיה"** היא אפליקציית ווב שמאפשרת צפייה בשידור חי של ערוצי טלוויזיה ישראליים ישירות מהדפדפן.
האפליקציה תומכת ב-5 ערוצים ישראליים ונבנתה כ-PWA (Progressive Web App) כך שניתן להתקין אותה כאפליקציה על הטלפון.

**פותח על ידי:** חיים רייך

### הערוצים הנתמכים

| ערוץ | מספר | מזהה | סוג סטרים |
|------|-------|-------|-----------|
| כאן 11 | 11 | `kan11` | HLS ישיר |
| קשת 12 | 12 | `keshet12` | HLS עם טוקן דינמי (AES-192-CBC) |
| רשת 13 | 13 | `reshet13` | HLS ישיר |
| ערוץ 14 | 14 | `channel14` | HLS ישיר |
| ערוץ הכנסת | 99 | `knesset` | HLS ישיר |

## ארכיטקטורה כללית

האפליקציה עובדת בצורה הבאה:

1. **המשתמש נכנס לאתר** - Netlify מגיש את הקבצים הסטטיים (HTML, JS, CSS)
2. **בחירת ערוץ** - המשתמש לוחץ על ערוץ ברשת הערוצים
3. **טעינת סטרים** - האפליקציה טוענת את ה-HLS stream מה-CDN של הערוץ
4. **ניגון וידאו** - הספרייה `hls.js` מנגנת את הסטרים בדפדפן
5. **מנגנון fallback** - אם ה-HLS נכשל, יש גיבוי לאתר הרשמי של הערוץ דרך iframe

### עיקרון מרכזי: כל ערוץ = כתובת HLS

כל הערוצים הישראליים משדרים בפרוטוקול **HLS** (HTTP Live Streaming) - פרוטוקול סטרימינג שמפרק את הווידאו לקטעים קטנים (segments) ומגיש אותם דרך playlist בפורמט `.m3u8`. הדפדפן טוען את ה-playlist, ואז מנגן את הקטעים ברצף.

**הרעיון הפשוט:** כל מה שצריך כדי לצפות בערוץ זה כתובת URL אחת של קובץ `.m3u8`. האתגר היה **למצוא את הכתובות הנכונות** של כל ערוץ.

## קשת 12 - התהליך המורכב

קשת 12 (מאקו) הוא הערוץ היחיד שדורש תהליך מורכב כדי לקבל את כתובת הסטרים. הסטרים מוגן מאחורי הצפנת AES-192-CBC ומערכת טוקנים מוגבלת בזמן.

### למה זה מורכב?

בניגוד לשאר הערוצים שפשוט יש להם כתובת HLS קבועה, קשת 12 מגינה על הסטרים שלה:
1. כתובת הסטרים **מוצפנת** ב-AES-192-CBC
2. כדי לנגן צריך **טוקן אימות** (hdnea של Akamai) שפג תוקף כל כמה דקות
3. גם בקשת הטוקן וגם התשובה **מוצפנות** ב-AES

### התהליך צעד אחר צעד

```
שלב 1: שליפת ה-Playlist המוצפן
═══════════════════════════════
GET https://www.mako.co.il/AjaxPage?jspName=playlist12.jsp&vcmid=...

התשובה: מחרוזת Base64 מוצפנת ב-AES-192-CBC

     ↓ פענוח

שלב 2: פענוח ה-Playlist
════════════════════════
מפתח: PLAYLIST_KEY = 'LTf7r/zM2VndHwP+4So6bw=='
IV:    'theExact16Chars='
אלגוריתם: AES-192-CBC

     ↓ JSON שמכיל את כתובת הבסיס של הסטרים

שלב 3: בקשת טוקן אימות
═══════════════════════
בניית payload:
{
  "lp": "/liveedge/hls/...",     ← נתיב הסטרים
  "rv": "AKAMAI",
  "dId": "<מזהה מכשיר אקראי>",
  "et": "egt"
}

הצפנת ה-payload ב-AES-192-CBC עם:
מפתח: TOKEN_KEY = 'YhnUaXMmltB6gd8p9SWleQ=='

POST https://mass.mako.co.il/ClicksStatistics/entitlementsServicesV2.jsp?et=egt
Body: Base64(AES_Encrypt(payload))

     ↓ תשובה מוצפנת

שלב 4: פענוח תשובת הטוקן
═════════════════════════
פענוח AES-192-CBC של התשובה

     ↓ JSON שמכיל:
     { "tickets": [{ "ticket": "hdnea=exp=...~acl=...~hmac=..." }] }

שלב 5: הרכבת הכתובת הסופית
════════════════════════════
URL סופי = כתובת_סטרים + "&" + ticket

     ↓

כתובת HLS מוכנה לניגון! ⏱️ (בתוקף לכמה דקות)
```

### מימוש AES-192-CBC מאפס

**אתגר טכני:** ה-Web Crypto API של הדפדפן **לא תומך ב-AES-192**. הוא תומך רק ב-AES-128 ו-AES-256.

**הפתרון:** מימוש **מלא** של AES-192-CBC ב-JavaScript טהור, כולל:
- S-Box ו-Inverse S-Box (טבלאות החלפה)
- Key Expansion (הרחבת מפתח ל-8 round keys)
- Galois Field Multiplication (כפל בשדה גלואה)
- SubBytes, ShiftRows, MixColumns, AddRoundKey (שלבי ההצפנה)
- CBC Mode (שרשור בלוקים עם IV)
- PKCS7 Padding

הקוד הזה נמצא בשני מקומות:
- **צד לקוח:** `src/lib/keshet12.ts` (TypeScript, 208 שורות)
- **צד שרת:** `worker/index.js` (JavaScript, חלק מ-394 שורות)

### מערכת Cache

התוצאה נשמרת ב-cache בזיכרון. ה-cache חכם - הוא קורא את ה-`exp=` parameter מתוך ה-hdnea token ויודע מתי הטוקן פג תוקף (עם מרווח ביטחון של 60 שניות). טוקן חדש מתבקש רק כשהישן פג.

```typescript
let cached: Keshet12Result | null = null;
// נבדק לפני כל בקשה - אם הטוקן עדיין בתוקף, מוחזר מה-cache
```

## מערכת הנגן

### סדר עדיפויות הניגון

```
1. HLS דרך hls.js (רוב הדפדפנים)
        ↓ אם הדפדפן לא תומך ב-MediaSource
2. HLS native (Safari/iOS - יש תמיכה מובנית)
        ↓ אם הסטרים נכשל
3. ניסיון חוזר עם resolveUrl (לקשת 12 - רענון טוקן)
        ↓ אם עדיין נכשל
4. ניסיון עם fallbackUrl (אם קיים)
        ↓ אם גם זה נכשל
5. מעבר ל-iframe של האתר הרשמי
        ↓ אם אין fallback
6. הצגת הודעת שגיאה עם כפתור "נסה שוב"
```

### הגדרות hls.js

```typescript
new Hls({
  enableWorker: true,      // הרצה ב-Web Worker לביצועים טובים
  lowLatencyMode: true,    // מצב השהייה נמוכה
  backBufferLength: 30,    // 30 שניות buffer אחורי
})
```

### סנכרון חזרה לשידור חי

כשהמשתמש חוזר לטאב אחרי שעזב, הנגן **קופץ אוטומטית לנקודה החיה** (live edge):

```typescript
// כשהטאב הופך שוב לגלוי
document.addEventListener('visibilitychange', () => {
  if (document.visibilityState === 'visible') {
    hls.currentTime = hls.liveSyncPosition; // קפיצה לשידור חי
    video.play();
  }
});
```

## ניהול מצב

האפליקציה משתמשת ב-**Zustand** - ספריית state management קלה ל-React.

```typescript
// src/stores/tvStore.ts

interface TvState {
  channels: Channel[]        // כל הערוצים (5)
  currentChannel: Channel    // הערוץ הנבחר (ברירת מחדל: כאן 11)
  isLoading: boolean         // האם הסטרים בטעינה
  error: string | null       // הודעת שגיאה (או null)

  setChannel(channel)        // בחירת ערוץ
  setLoading(loading)        // עדכון מצב טעינה
  setError(error)            // עדכון שגיאה
  nextChannel()              // מעבר לערוץ הבא (מעגלי)
  prevChannel()              // מעבר לערוץ הקודם (מעגלי)
}
```

**ניווט מעגלי:** `nextChannel` ו-`prevChannel` עוברים בלולאה - מהערוץ האחרון חוזרים לראשון, ולהפך. זה משמש גם את פקדי מסך הנעילה (הבא/הקודם).

## Hooks מותאמים אישית

### useMediaSession - שליטה ממסך נעילה

```
מאפשר שליטה מממשק מסך הנעילה / אזור ההתראות:
- שם הערוץ מוצג ככותרת
- "שידור חי" כאמן
- "טלוויזיה ישראלית" כאלבום
- לוגו הערוץ כתמונה
- כפתורי הבא/הקודם → nextChannel/prevChannel
- כפתורי נגן/עצור → toggle play/pause
```

### usePiP - Picture-in-Picture

```
מאפשר חלון וידאו צף:
- תמיכה ב-requestPictureInPicture() (סטנדרט)
- fallback ל-webkitSetPresentationMode (Safari)
- זמין רק במצב HLS (לא iframe)
```

### useWakeLock - מניעת כיבוי מסך

```
מונע מהמסך להיכבות בזמן צפייה:
- משתמש ב-Screen Wake Lock API
- מתחדש אוטומטית כשחוזרים לטאב
- נכשל בחן בדפדפנים שלא תומכים
```

## PWA

האפליקציה בנויה כ-**Progressive Web App** מלאה:

### יכולות PWA

| תכונה | פירוט |
|--------|--------|
| **התקנה** | ניתן להתקין כאפליקציה על טלפון/מחשב |
| **מצב standalone** | נפתחת ללא סרגלי דפדפן |
| **Service Worker** | עדכון אוטומטי ברקע |
| **cache offline** | קבצים סטטיים נשמרים ב-cache |
| **RTL** | כיוון ימין-לשמאל בהגדרות ה-manifest |
| **ערכת נושא** | רקע כהה (#0a0a0a) |
| **אייקונים** | SVG בגדלים 192x192 ו-512x512 (כולל maskable) |

### אסטרטגיית Cache

```
קבצים סטטיים (JS, CSS, HTML, SVG, PNG, WOFF2):
  → Precache בזמן התקנת ה-Service Worker

Google Fonts:
  → CacheFirst עם TTL של שנה

ניווט SPA:
  → Fallback ל-/index.html
```

## סיכום

### מה הפרויקט עושה
אפליקציית ווב שמאפשרת צפייה בשידור חי של 5 ערוצי טלוויזיה ישראליים ישירות מהדפדפן, עם אפשרות התקנה כאפליקציה.

### איך הוא עושה את זה
1. **מציאת כתובות סטרים** - איתור כתובות HLS (.m3u8) של כל ערוץ מה-CDN-ים שלהם
2. **ניגון בדפדפן** - שימוש ב-hls.js לניגון סטרימי HLS
3. **פיצוח קשת 12** - מימוש AES-192-CBC מאפס כדי לפענח את ה-API המוצפן של מאקו ולקבל טוקנים
4. **חוויית מובייל** - PWA, Wake Lock, Media Session, PiP
5. **מנגנון גיבוי** - אם HLS נכשל → iframe של האתר הרשמי

### הישגים טכניים עיקריים
- **מימוש AES-192-CBC ב-JavaScript טהור** - כי Web Crypto API לא תומך ב-AES-192
- **מנגנון fallback רב-שכבתי** - HLS → רענון טוקן → HLS חלופי → iframe → שגיאה
- **PWA מלא** - התקנה, cache offline, Service Worker עם עדכון אוטומטי
- **שילוב APIs מתקדמים** - Media Session, Wake Lock, PiP, Fullscreen
- **אפס תלויות UI** - עיצוב מלא עם Tailwind בלבד, אייקונים SVG inline
- **קוד קצר ויעיל** - ~15 קבצי מקור, ללא ספריות מיותרות
